Create a detailed implementation plan for: $ARGUMENTS

## Workflow

### 1. ANALYZE CURRENT STATE
- Read relevant files to understand context
- Identify existing patterns and conventions
- Assess current codebase structure

### 2. REQUIREMENTS
- Extract explicit requirements from request
- Identify implicit requirements
- Note constraints and limitations

### 3. PLAN
Break down into:
- Step-by-step implementation sequence
- Files to create/modify
- Dependencies between steps
- Testing strategy for each step

### 4. CHALLENGES
- Potential issues and blockers
- Solutions for each challenge
- Risk mitigation

## Output Format

```
## Implementation Plan

### Current State
[Assessment of existing code]

### Requirements
- [Requirement 1]
- [Requirement 2]

### Implementation Steps
| Step | Action | File | Notes |
|------|--------|------|-------|
| 1 | [Action] | file.js | [Note] |

### Testing Strategy
- [Test approach for each step]

### Challenges & Solutions
| Challenge | Solution |
|-----------|----------|
| [Challenge] | [Solution] |
```
